# Levis AI機能ガイド

Levisには執筆専用のAI Agentが内蔵されています。このドキュメントでは、サインインの方法、AI機能の使い方、そして「Agentワークスペース」を使って自分だけの執筆アシスタントにカスタマイズする方法を説明します。

> [!TIP]
> すべてのAI機能は **設定 > AI** で個別にオン/オフできます。ここには全プロバイダーの一覧もあり、「使用する」ボタンで切り替えられます。

## サインイン / プロバイダーの設定

**設定 > AI** はプロバイダーごとに展開できる一覧になっており、クリックしてサインイン・設定し、「使用する」で切り替えます:

| プロバイダー | 説明 |
| --- | --- |
| **ChatGPT (Codex)** | ChatGPTアカウントでサインイン(ブラウザ認証、APIキー不要)。ツール呼び出し(編集の提案、ファイル読み込み、スキルの動的読み込み)とWeb検索に対応。 |
| **OpenAI APIキー** | 標準的なOpenAI APIキーを貼り付けます。ChatGPTと同じくツール呼び出しとWeb検索に対応。 |
| **Claude** | Claudeアカウントでサインイン(ブラウザ認証)。ツール呼び出し(編集の提案、ファイル読み込み、スキルの動的読み込み)に対応、Web検索は未対応。 |
| **カスタムエンドポイント** | OpenAI互換の任意のエンドポイント(ローカルモデル、中継サービスなど):エンドポイントアドレスと任意のキーを入力し、モデル名を取得するか手入力します。OpenRouter、Groq、ローカルのOllamaサーバー向けのクイック入力ボタンがあります。ツール呼び出しはベストエフォートです——すべての互換サーバーが正しく対応しているわけではないため、失敗した場合は編集を提案する代わりに自動的にテキストのみの返答にフォールバックします。 |

「ChatGPTでサインイン」/「Claudeでサインイン」をクリックするとブラウザが開いて認証が行われ、アプリに戻れば完了です。

## AI機能

### 1. 続き書き(補完)

入力中にカーソル位置へ自動的にグレーの続き書き候補が表示され、<kbd>Tab</kbd>で採用します。最小文字数の制限はなく、文書の最初の一文からでも動作します。<kbd>Cmd</kbd>+<kbd>Shift</kbd>+<kbd>Space</kbd> または右クリックメニューから手動で呼び出すこともできます。

**設定 > AI** で**補完のトーン**（文書に合わせる / フォーマル / カジュアル / 学術的 / 簡潔）を調整できます。

### 2. 文法チェック

問題の可能性がある箇所に下線が引かれ、ホバーすると説明が表示され、ワンクリックで修正を適用できます。ショートカット <kbd>Cmd</kbd>+<kbd>Shift</kbd>+<kbd>G</kbd> で現在の段落を即座にチェックします。

### 3. AIに聞く(対話ポップアップ)

<kbd>Cmd</kbd>+<kbd>Shift</kbd>+<kbd>K</kbd>(または右クリック > AIに聞く)でカーソル位置に対話を開きます——呼び出した位置に固定された1本の入力バーで、会話が進むにつれてその上に積み上がっていき、入力欄自体が下に押し出されることはありません。文書全体を把握しており、開く前にテキストを選択していれば、その選択範囲が自動的にコンテキストとして使われます。

- **履歴**：サイドバーの「チャット」タブに過去の会話が保存されます。クリックすると現在のエディタで開き直して続けられます。
- **ファイルの添付**:入力欄左側の **+** をクリックすると、任意のテキストファイル(資料、アウトライン、別の章など)をそのメッセージに添付できます。
- **Web検索**:設定で有効にすると(ChatGPTとOpenAI APIキープロバイダー)、必要に応じてAgentが自らWebを検索します。
- **ワークスペース内ファイルの読み込み**:文書が保存済みの場合、ツール呼び出しに対応したプロバイダー(ChatGPT、APIキー、Claude)は文書があるフォルダ内のファイルを自分で一覧・読み込みできます —— 参考資料を文書のそばに置くだけで利用できます。

返信はあくまでテキストによる説明であり、対話自体が文書を直接書き換えることはありません。文書を変更する返信は、必ず文書内編集プレビュー(次の項目)として現れます。

### 4. 文書内での編集プレビュー

ツール呼び出しに対応したプロバイダー(ChatGPT、OpenAI APIキー、Claude、またはツール呼び出しに対応したカスタムエンドポイント)に文書の変更を依頼すると、変更箇所にその場でプレビューが表示されます:削除されるテキストには取り消し線、追加されるテキストは緑色の幽霊テキストとして表示されます。すぐそばに **✓ 承認** / **✗ 却下** の小さなパネルが浮かび、ショートカット <kbd>Cmd</kbd>+<kbd>Return</kbd> / <kbd>Cmd</kbd>+<kbd>Delete</kbd> でも同じ操作ができます。承認するまで文書には一切変更が加わらず、<kbd>Cmd</kbd>+<kbd>Z</kbd>で承認済みの変更を一度で取り消せます。対話パネルのカードにも同じ承認/却下ボタンがあり、処理後はステータスが表示されます。返信が文書を変更できるのはこの方法だけで、自分で挿入・置換先を選ぶボタンはありません。

1回の返信に複数の編集提案がある場合は同時にプレビューされます。フローティングパネルには「n件中i件目」と表示され、‹›で切り替え、すべて承認/すべて却下もできます。

## Agentワークスペース

執筆プロジェクトごとに必要な指示、スキル、資料は異なります。ワークスペースとは**文書があるフォルダ**のことで、その中に `.levis/` ディレクトリを置くことでAgentをカスタマイズできます:

```
私の小説/
├── 第一章.md          ← 編集中の文書
├── .levis/
│   ├── agent.md       ← 常設の指示:文体、用語集、プロジェクトの背景
│   └── skills/
│       ├── polish.md  ← スキル:1ファイル1スキル
│       └── outline.md
└── 資料/
    └── 設定.md         ← Agentが自分で読み込める参考資料
```

さらに、すべての文書に適用される**グローバル層**もあります（設定 > Agent > グローバルフォルダを開く）。構造は同じです。ワークスペース内の同名スキルはグローバルのものを上書きします。

### agent.md — 常設の指示

`agent.md` の内容は毎回の会話でAgentに伝えられます。文体の要件、人名・地名などの用語集、このプロジェクトが何なのか、といった内容を書くのに適しています。

### skills/ — スキル

スキル = 1つのMarkdownファイル:先頭にYAMLの説明、本文に完全な指示を書きます。

```markdown
---
name: polish
description: 選択した段落を、意味を変えずに引き締めて磨き上げる
---

与えられたテキストを一文ずつ磨き上げる:冗長な語を削り、
長い文を短く分割し、元の意味とトーンは変えない。
磨き上げた後のテキストのみを出力する。
```

スキルの呼び出し方法は2つあります:

1. **手動**:対話ボックスに `/` と入力し、メニューから選ぶ(または直接 `/polish この段落も見て` のように入力)。
2. **自動**(ツール呼び出しに対応したプロバイダーのみ):Agentは各スキルの名前と説明を把握しており、あなたの依頼がいずれかのスキルの説明と一致すると、自動的にそのスキルを読み込んで完全な指示に従います。

> [!NOTE]
> スキルファイルは編集するとすぐに反映されます —— 次に対話ボックスを開いたときには新しい内容が使われ、再起動は不要です。

## プライバシーについて

文書の内容、添付したファイル、ワークスペースの指示は、選択したプロバイダーへリクエストとともに送信されます。Web検索はOpenAIのサーバー側で実行されます。不要な場合は設定で該当機能をオフにしてください。
