# Levis AI Agent ガイド

Levisには執筆専用のAI Agentが内蔵されています。このドキュメントでは、サインインの方法、3つのAI機能の使い方、そして「Agentワークスペース」を使って自分だけの執筆アシスタントにカスタマイズする方法を説明します。

> [!TIP]
> すべてのAI機能は **設定 > AI** で個別にオン/オフできます。

## サインイン / プロバイダーの設定

**設定 > AI > プロバイダー** で方式を選びます:

| プロバイダー | 説明 |
| --- | --- |
| **ChatGPT (Codex)** | ChatGPTアカウントでサインイン(ブラウザ認証、APIキー不要)。最も機能が充実しており、ツール呼び出し(編集の提案、ファイル読み込み、スキルの動的読み込み、Web検索)に対応するのはこのプロバイダーのみです。 |
| **Claude** | Claudeアカウントでサインイン(ブラウザ認証)。 |
| **OpenAI APIキー** | 標準的なOpenAI APIキーを貼り付けます。 |
| **カスタムエンドポイント** | OpenAI互換の任意のエンドポイント(ローカルモデル、中継サービスなど):エンドポイントアドレスと任意のキーを入力し、モデル名を取得するか手入力します。 |

「ChatGPTでサインイン」/「Claudeでサインイン」をクリックするとブラウザが開いて認証が行われ、アプリに戻れば完了です。

## 3つのAI機能

### 1. 続き書き(補完)

入力中にカーソル位置へ自動的にグレーの続き書き候補が表示され、<kbd>Tab</kbd>で採用します。<kbd>Cmd</kbd>+<kbd>Shift</kbd>+<kbd>Space</kbd> または右クリックメニューから手動で呼び出すこともできます。

**設定 > AI** で**補完のトーン**(フォーマル / カジュアル / 学術的 / 簡潔)を調整でき、**補完の指示**(例:「ダッシュを使わない」)を書いておくと、毎回の補完リクエストに追加されます。

### 2. 文法チェック

問題の可能性がある箇所に下線が引かれ、ホバーすると説明が表示され、ワンクリックで修正を適用できます。ショートカット <kbd>Cmd</kbd>+<kbd>Shift</kbd>+<kbd>G</kbd> で現在の段落を即座にチェックします。

### 3. AIに聞く(対話Agent)

<kbd>Cmd</kbd>+<kbd>Shift</kbd>+<kbd>K</kbd>(または右クリック > AIに聞く)でカーソル位置にダイアログを開きます。文書全体を把握しており、開く前にテキストを選択していれば、その選択範囲が自動的にコンテキストとして使われます。

- **編集の提案**:文書の変更を依頼すると(ChatGPTプロバイダー)、追加・削除を対比表示する編集カードが提示されます。「適用」を押して初めて文書に反映され、<kbd>Cmd</kbd>+<kbd>Z</kbd>で元に戻せます。
- **ファイルの添付**:入力欄左側の **+** をクリックすると、任意のテキストファイル(資料、アウトライン、別の章など)をそのメッセージに添付できます。
- **Web検索**:設定で有効にすると(ChatGPTプロバイダーのみ)、必要に応じてAgentが自らWebを検索します。
- **ワークスペース内ファイルの読み込み**:文書が保存済みの場合(ChatGPTプロバイダーのみ)、Agentは文書があるフォルダ内のファイルを自分で一覧・読み込みできます —— 参考資料を文書のそばに置くだけで利用できます。

## Agentワークスペース

執筆プロジェクトごとに必要な指示、スキル、資料は異なります。ワークスペースとは**文書があるフォルダ**のことで、その中に `.levis/` ディレクトリを置くことでAgentをカスタマイズできます:

```
私の小説/
├── 第一章.md          ← 編集中の文書
├── .levis/
│   ├── agent.md       ← 常設の指示:文体、用語集、プロジェクトの背景
│   └── skills/
│       ├── polish.md  ← スキル:1ファイル1スキル
│       └── outline.md
└── 資料/
    └── 設定.md         ← Agentが自分で読み込める参考資料
```

さらに、すべての文書に適用される**グローバル層**もあります(設定 > AI > Agentワークスペース > グローバルフォルダを開く)。構造は同じです。ワークスペース内の同名スキルはグローバルのものを上書きします。

### agent.md — 常設の指示

`agent.md` の内容は毎回の会話でAgentに伝えられます。文体の要件、人名・地名などの用語集、このプロジェクトが何なのか、といった内容を書くのに適しています。

### skills/ — スキル

スキル = 1つのMarkdownファイル:先頭にYAMLの説明、本文に完全な指示を書きます。

```markdown
---
name: polish
description: 選択した段落を、意味を変えずに引き締めて磨き上げる
---

与えられたテキストを一文ずつ磨き上げる:冗長な語を削り、
長い文を短く分割し、元の意味とトーンは変えない。
磨き上げた後のテキストのみを出力する。
```

スキルの呼び出し方法は2つあります:

1. **手動**:対話ボックスに `/` と入力し、メニューから選ぶ(または直接 `/polish この段落も見て` のように入力)。
2. **自動**(ChatGPTプロバイダーのみ):Agentは各スキルの名前と説明を把握しており、あなたの依頼がいずれかのスキルの説明と一致すると、自動的にそのスキルを読み込んで完全な指示に従います。

> [!NOTE]
> スキルファイルは編集するとすぐに反映されます —— 次に対話ボックスを開いたときには新しい内容が使われ、再起動は不要です。

## プライバシーについて

文書の内容、添付したファイル、ワークスペースの指示は、選択したプロバイダーへリクエストとともに送信されます。Web検索はOpenAIのサーバー側で実行されます。不要な場合は設定で該当機能をオフにしてください。
