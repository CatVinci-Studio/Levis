# Levis AI 功能指南

Levis 内置一个写作专用的 AI Agent。这份文档介绍如何登录、AI 功能怎么用，以及如何用「Agent 工作区」把它定制成你自己的写作助手。

> [!TIP]
> 所有 AI 功能都可以在 **设置 > AI** 里单独开关，那里也列出了全部 provider，点「使用」即可切换。

## 登录 / 配置提供方

**设置 > AI** 里是一份可以展开的 provider 列表，点击展开登录/配置，点「使用」切换：

| 提供方 | 说明 |
| --- | --- |
| **ChatGPT (Codex)** | 用你的 ChatGPT 账号登录（浏览器授权，无需 API Key）。支持工具调用（修改提议、读文件、技能动态加载）与联网搜索。 |
| **OpenAI API Key** | 粘贴一个标准的 OpenAI API Key。与 ChatGPT 一样支持工具调用和联网搜索。 |
| **Claude** | 用你的 Claude 账号登录（浏览器授权）。支持工具调用（修改提议、读文件、技能动态加载），暂不支持联网搜索。 |
| **自定义接口** | 任何 OpenAI 兼容的接口（如本地模型、中转服务）：填接口地址、可选的 Key，然后获取或手填模型名。提供 OpenRouter、Groq 和本地 Ollama 的快速填入按钮。工具调用是尽力而为——不是每个兼容服务器都能正确支持，请求失败时会自动降级为纯文字回复，而不是提出修改。 |

点击「使用 ChatGPT 登录」/「使用 Claude 登录」后会打开浏览器完成授权，回到应用即完成。

## AI 功能

### 1. 续写（补全）

输入时自动在光标处给出灰色的续写建议，按 <kbd>Tab</kbd> 采纳。没有最小字数限制，文档第一句话就能触发。也可以用 <kbd>Cmd</kbd>+<kbd>Shift</kbd>+<kbd>Space</kbd> 或右键菜单手动触发。

在 **设置 > AI** 里可以调整**续写语气**（匹配文档 / 正式 / 轻松 / 学术 / 简洁）。

### 2. 语法检查

对可能有问题的文字加下划线，悬停查看说明并一键应用修改。快捷键 <kbd>Cmd</kbd>+<kbd>Shift</kbd>+<kbd>G</kbd> 立即检查当前段落。

### 3. 询问 AI（对话弹窗）

按 <kbd>Cmd</kbd>+<kbd>Shift</kbd>+<kbd>K</kbd>（或右键 > 询问 AI）在光标处打开对话——一条固定在光标附近的输入条，随着对话的进行往上叠加展开，而不会把输入位置往下推。它了解整篇文档；如果打开前选中了文字，选中内容会自动作为上下文。

- **历史记录**：侧栏的「对话」标签页会保存过去的对话，点击其中一条即可在当前编辑器里重新打开并继续。
- **附加文件**：点输入框左侧的 **+** 可以把任意文本文件附到这条消息里（例如资料、大纲、另一章）。
- **联网搜索**：在设置里开启后（ChatGPT 和 OpenAI API Key 提供方），Agent 需要时可以自己搜索网页。
- **读工作区文件**：文档已保存时，支持工具调用的提供方（ChatGPT、API Key 或 Claude）可以自己列出并阅读文档所在文件夹里的文件——把参考资料放在文档旁边即可。

回复本身只是文字说明，对话框不会直接改动文档——任何要改文档的回复都会以文档内编辑预览的形式出现（见下一节）。

### 4. 文档内编辑预览

当你让支持工具调用的提供方（ChatGPT、OpenAI API Key、Claude，或支持工具调用的自定义接口）修改文档时，修改会直接呈现在原位：删除的文字加删除线，新增的文字以绿色幽灵文本显示。旁边会浮现一个小面板，**✓ 采纳** / **✗ 拒绝**，快捷键 <kbd>Cmd</kbd>+<kbd>Return</kbd> / <kbd>Cmd</kbd>+<kbd>Delete</kbd> 效果相同。采纳前文档不会有任何变化，<kbd>Cmd</kbd>+<kbd>Z</kbd> 可以一步撤销已采纳的修改。对话卡片上也有同样的采纳/拒绝按钮，处理后会显示状态。这是回复能改动文档的唯一方式——没有另外的"插入"或"替换"按钮需要你自己决定放在哪里。

一条回复里有多个修改提议时会同时预览；浮动面板显示「第 i 个，共 n 个」并可用 ‹› 切换，还有全部采纳/全部拒绝。

## Agent 工作区

不同的写作项目需要不同的指令、技能和资料。工作区就是**文档所在的文件夹**，在里面放一个 `.levis/` 目录即可定制 Agent：

```
我的小说/
├── 第一章.md          ← 正在编辑的文档
├── .levis/
│   ├── agent.md       ← 常驻指令：文风、术语表、项目背景
│   └── skills/
│       ├── polish.md  ← 技能：一个文件一个技能
│       └── outline.md
└── 资料/
    └── 设定.md         ← Agent 可以自己读取的参考资料
```

另外还有一个**全局层**，对所有文档生效（设置 > Agent > 打开全局文件夹），结构相同。工作区里同名的技能会覆盖全局的。

### agent.md — 常驻指令

`agent.md` 里的内容每次对话都会告诉 Agent。适合放：写作风格要求、人名地名术语表、这个项目是什么。

### skills/ — 技能

一个技能 = 一个 Markdown 文件：开头是 YAML 说明，正文是完整指令。

```markdown
---
name: polish
description: 润色选中的段落，保持原意，收紧句子
---

对给出的文字逐句润色：删掉冗余的词，把长句拆短，
保持原本的意思和语气不变。只输出润色后的文字。
```

技能有两种触发方式：

1. **手动**：在对话框输入 `/`，从菜单里选（或直接输 `/polish 再帮我看看这段`）。
2. **自动**（仅支持工具调用的提供方）：Agent 知道每个技能的名字和描述，当你的请求和某个技能的描述匹配时，它会自己加载并遵循完整指令。

> [!NOTE]
> 技能文件改完即生效——下次打开对话框就会用新的内容，无需重启。

## 隐私说明

文档内容、附加的文件和工作区指令会随请求发送给你所选的提供方。联网搜索由 OpenAI 在服务端执行。不需要时在设置里关掉对应功能即可。
